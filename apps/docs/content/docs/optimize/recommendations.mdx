Optimize provides recommendations focused on performance improvements such as indexing issues, excessive data retrieval, and inefficient query patterns. Recommendations include:

- [Excessive number of rows returned](/docs/v7/postgres/query-optimization/recommendations#excessive-rows-returned)
- [Full table scans caused by LIKE operations](/docs/v7/postgres/query-optimization/recommendations#full-table-scans-from-like-operations)
- [Queries on unindexed columns](/docs/v7/postgres/query-optimization/recommendations#queries-on-unindexed-columns)
- [Repeated query](/docs/v7/postgres/query-optimization/recommendations#repeated-queries)
- [Overfetching](/docs/v7/postgres/query-optimization/recommendations#overfetching-data)
- [Using `@db.Money`](/docs/v7/postgres/query-optimization/recommendations#avoid-dbmoney)
- [Using `@db.Char(n)`](/docs/v7/postgres/query-optimization/recommendations#avoid-dbcharn)
- [Using `@db.VarChar(n)`](/docs/v7/postgres/query-optimization/recommendations#avoid-dbvarcharn)
- [Using `timestamp(0)` or `timestamptz(0)`](/docs/v7/postgres/query-optimization/recommendations#avoid-timestamp0-and-timestamptz0)
- [Using `CURRENT_TIME`](/docs/v7/postgres/query-optimization/recommendations#avoid-current_time)
- [Storing large objects or BLOBs in the database](/docs/v7/postgres/query-optimization/recommendations#storing-blobs-in-database)
- [Indexing on unique columns](/docs/v7/postgres/query-optimization/recommendations#indexing-on-unique-columns)
- [Long-running transactions](/docs/v7/postgres/query-optimization/recommendations#long-running-transactions)
- [Unnecessary indexes](/docs/v7/postgres/query-optimization/recommendations#unnecessary-indexes)

Learn more about the [recommendations generated by Optimize here](/docs/v7/postgres/query-optimization/recommendations).
