Prisma Client supports full-text search for **PostgreSQL** (v2.30.0+) and **MySQL** (v3.8.0+, GA in v6).

## Enabling for PostgreSQL

```prisma
generator client {
  provider        = "prisma-client"
  previewFeatures = ["fullTextSearchPostgres"]
}
```

## Basic usage

```ts
const result = await prisma.posts.findMany({
  where: { body: { search: "cat" } },
});
```

## Query syntax

### PostgreSQL

| Query | Description |
|-------|-------------|
| `fox & dog` | Contains 'fox' AND 'dog' |
| `fox \| cat` | Contains 'fox' OR 'cat' |
| `!cat` | Does NOT contain 'cat' |
| `fox <-> dog` | 'dog' follows 'fox' |

```ts
const result = await prisma.posts.findMany({
  where: { body: { search: "cat & dog" } },
});
```

### MySQL

| Query | Description |
|-------|-------------|
| `+fox +dog` | Contains 'fox' AND 'dog' |
| `fox dog` | Contains 'fox' OR 'dog' |
| `+dog -cat` | Contains 'dog' but NOT 'cat' |
| `quic*` | Contains word starting with 'quic' |

```ts
const result = await prisma.posts.findMany({
  where: { body: { search: "+cat -dog" } },
});
```

## Sorting by relevance

```ts
const posts = await prisma.post.findMany({
  orderBy: {
    _relevance: {
      fields: ["title"],
      search: "database",
      sort: "asc",
    },
  },
});
```

## MySQL indexes

For MySQL, add `@@fulltext` indexes to searchable columns:

```prisma
model Blog {
  id      Int    @unique
  content String
  title   String

  @@fulltext([content])
  @@fulltext([content, title])
}
```

## Raw SQL for PostgreSQL

For better performance, use TypedSQL:

```sql
SELECT * FROM "Blog" WHERE to_tsvector('english', "Blog"."content") @@ to_tsquery('english', ${term});
```
