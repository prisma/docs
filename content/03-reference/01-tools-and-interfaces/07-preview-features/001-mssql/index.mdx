### Data model limitations

#### Cannot use column with `UNIQUE` constraint and filtered index as foreign key

MSSQL [only allows one `NULL` value in a column that has a `UNIQUE` constraint](https://docs.microsoft.com/en-us/sql/relational-databases/tables/unique-constraints-and-check-constraints?view=sql-server-ver15#Unique). For example:

* A table of users has a column named `license_number`
* The `license_number` field has a `UNIQUE` constraint
* The `license_number` field only allows **one** `NULL` value

The standard way to get around this issue is to create a filtered unique index that excludes `NULL` values. This allows you to insert multiple `NULL` values.

*However*, this setup makes it impossible to use `license_number` as a foreign key.

### Connection string limitations

* Cannot use the following format for connection string MYSERVER\SQL2019 - fails validation
* It is possible to use escaping and have the delimiter character ; in the values, such as in the password. This should be written as {;}, but for now we fail bad when these arrive in the string.
