<TopBlock>

You can enable debugging output in Prisma Client via the [`DEBUG`](/reference/api-reference/environment-variables-reference#debug) environment variable. It can be configured with the following values:

- `prisma:engine`: Prints relevant debug messages happening in a Prisma [engine](https://github.com/prisma/prisma-engines/)
- `prisma:client`: Prints relevant debug messages happening in the Prisma Client runtime
- `prisma*`: Prints all debug messages from Prisma Client or CLI
- `*`: Prints all debug messages

It's also possible to provide multiple values separated by a comma, for example: `prisma:engine,prisma:client`.

<Admonition type="info">

Prisma Client can be configured to log warnings, errors and information related to queries sent to the database. See [Configuring logging](./working-with-prismaclient/logging) for more information.

</Admonition>

</TopBlock>

## Setting the `DEBUG` environment variable through the command line

Here are examples for setting these debugging options through the command line:

<TabbedContent tabs={[<FileWithIcon text="Bash (Linux & macOS)" icon="code"/>, <FileWithIcon text="Windows" icon="code"/>]}>
<tab>

```terminal
# enable only `prisma:engine`-level debugging output
export DEBUG="prisma:engine"

# enable both `client` and `engine` level debugging output
export DEBUG="prisma:client,prisma:engine"

# enable all debugging output
export DEBUG="*"
```

</tab>
<tab>

```terminal
# enable only `prisma:engine`-level debugging output
set DEBUG=prisma:engine

# enable both `client` and `engine` level debugging output
set DEBUG=prisma:client,prisma:engine

# enable all debugging output
set DEBUG=*
```

</tab>
</TabbedContent>

## Setting the `DEBUG` environment variable inside your `.env` file

Any environment variables defined in that `.env` file will automatically be loaded when running a Prisma CLI command.

```terminal file=.env
DEBUG="prisma*"
```

The Prisma CLI automatically loads `.env` files, but the Prisma Client does not. You can use a third-party library like [`dotenv`](https://www.npmjs.com/package/dotenv) to preload environment variables from a `.env` file before running your application.

For example, if you want to run `index.js` or `index.ts`, you can preload variables from `.env` and execute it with the following command:

<TabbedContent tabs={[<FileWithIcon text="JavaScript" icon="code"/>, <FileWithIcon text="TypeScript" icon="code"/>]}>
<tab>

```terminal
node -r dotenv/config index.js
```

</tab>
<tab>

```terminal
ts-node -r dotenv/config index.ts
```

</tab>
</TabbedContent>
