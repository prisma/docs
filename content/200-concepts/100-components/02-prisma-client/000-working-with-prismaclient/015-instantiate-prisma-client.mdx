## Use a single shared instance of <inlinecode>PrismaClient</inlinecode>

In most cases, we recommend that you create and use a single instance of `PrismaClient` across your application.

2. Create query engine
3. Create [connection pool](connection-management#connection-pool).

Too many instances of `PrismaClient` results in:

```
Error in connector: Error querying the database: db error: FATAL: sorry, too many clients already
    at PrismaClientFetcher.request
```

Simplest example:

```ts file=client.ts
import { PrismaClient } from "@prisma/client"

let prisma = new PrismaClient()

export default prisma
```

Use:

```ts file=app.ts
import prisma from './client'

async function main() {

    const allUsers = await prisma.user.findMany();
}

main();
```

Support dev environment:


```ts file=client.ts
import { PrismaClient } from "@prisma/client"

declare global {
    namespace NodeJS {
        interface Global {
            prisma: PrismaClient;
        }
    }
}

let prisma: PrismaClient;

if (process.env.NODE_ENV === "production") {
  prisma = new PrismaClient()
} 
else  
{
  if (!global.prisma) {
    global.prisma = new PrismaClient()
  }

  prisma = global.prisma
}

export default prisma
```

Use in the same way:

```ts file=app.ts
import prisma from './client'

async function main() {

    const allUsers = await prisma.user.findMany();
}

main();
```
