## Create a single shared instance of <inlinecode>PrismaClient</inlinecode>

Each time you create an instance of `PrismaClient`, the query engine creates a connection pool. If there are too many clients, you will get the following error:

```
Error in connector: Error querying the database: db error: FATAL: sorry, too many clients already
    at PrismaClientFetcher.request
```

One option is to make `PrismaClient` available as a global variable. The following example:

* Creates a new instance of `PrismaClient` on the first request
* Returns the same instance for subsequence requests

```ts
import { Prisma, PrismaClient } from '@prisma/client'

// Declare global variable
declare global {
    namespace NodeJS {
        interface Global {
            prisma: PrismaClient;
        }
    }
}

let prisma: PrismaClient;

// If global `prisma` property not set,
// instantiate client
if (!global.prisma)
{
    global.prisma = new PrismaClient();
}

// Return global prisma property
prisma = global.prisma;

export { prisma }
```
