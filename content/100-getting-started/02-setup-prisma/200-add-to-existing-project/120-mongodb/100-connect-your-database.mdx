## Connect your database

To connect your database, you need to set the `url` field of the `datasource` block in your Prisma schema to your database [connection URL](/reference/database-reference/connection-urls):

```prisma file=prisma/schema.prisma
datasource db {
  provider = "mongodb"
  url      = env("DATABASE_URL")
}
```

In this case, the `url` is [set via an environment variable](/guides/development-environment/environment-variables) which is defined in `.env`:

<SwitchTech technologies={['*', 'mongodb']}>

```bash file=.env
DATABASE_URL="mongodb://test:test@localhost:27017/myFirstDatabase/"
```

</SwitchTech>

<SwitchTech technologies={['*', 'atlas']}>

```bash file=.env
DATABASE_URL="mongodb+srv://test:test@cluster0.ns1yp.mongodb.net/myFirstDatabase"
```

</SwitchTech>

You now need to adjust the connection URL to point to your own database.

The format of the connection URL for your database depends on the database you use. For MongoDB, it looks as follows (the parts spelled all-uppercased are _placeholders_ for your specific connection details):

<SwitchTech technologies={['*', 'mongodb']}>

```no-lines
mongodb://USERNAME:PASSWORD@HOST:PORT/DATABASE/
```

Here's a short explanation of each component:

- `USERNAME`: The name of your database user
- `PASSWORD`: The password for your database user
- `HOST`: The host where a [`mongod`](https://docs.mongodb.com/manual/reference/program/mongod/#mongodb-binary-bin.mongod) (or [`mongos`](https://docs.mongodb.com/manual/reference/program/mongos/#mongodb-binary-bin.mongos)) instance is running
- `PORT`: The port where your database server is running (typically `27017` for MongoDB)
- `DATABASE`: The name of the database

As an example, for a MongoDB database hosted at localhost, the [connection URL](/reference/database-reference/connection-urls) might look similar to this:

```bash file=.env
DATABASE_URL="mongodb://test:test@localhost:27017/myFirstDatabase"
```

</SwitchTech>

<SwitchTech technologies={['*', 'atlas']}>

```no-lines
mongodb+srv://USERNAME:PASSWORD@HOST/DATABASE
```

Here's a short explanation of each component:

- `USERNAME`: The name of your database user
- `PASSWORD`: The password for your database user
- `HOST`: The host where a [`mongod`](https://docs.mongodb.com/manual/reference/program/mongod/#mongodb-binary-bin.mongod) (or [`mongos`](https://docs.mongodb.com/manual/reference/program/mongos/#mongodb-binary-bin.mongos)) instance is running
- `DATABASE`: The name of the database

As an example, for a MongoDB database hosted on [MongoDB Atlas](https://www.mongodb.com/cloud/atlas), the [connection URL](/reference/database-reference/connection-urls) might look similar to this:

```bash file=.env
DATABASE_URL="mongodb+srv://test:test@cluster0.ns1yp.mongodb.net/myFirstDatabase"
```

</SwitchTech>
