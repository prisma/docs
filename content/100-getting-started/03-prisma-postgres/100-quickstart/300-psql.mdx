[psql](https://www.postgresql.org/docs/current/app-psql.html) is the PostgreSQL interactive terminal, a command-line tool that enables you to type in queries interactively, issue them to PostgreSQL, and see the query results. In this guide, you'll learn how to connect to [Prisma Postgres](/postgres) using psql and start running SQL commands.

## Prerequisites

- **PostgreSQL client tools** - psql is included with PostgreSQL installations
- A [Prisma Data Platform](https://console.prisma.io) account

## 1. Install psql

If you don't have psql installed, you can install it:

**macOS (using Homebrew):**
```terminal
brew install postgresql
```

**Ubuntu/Debian:**
```terminal
sudo apt-get install postgresql-client
```

**Windows:**
Download and install PostgreSQL from the [official website](https://www.postgresql.org/download/windows/), which includes psql.

Verify the installation:
```terminal
psql --version
```

## 2. Create a Prisma Postgres database

Follow these steps to create your Prisma Postgres database:

1. Log in to [Prisma Data Platform](https://console.prisma.io) and open the Console.
2. In a workspace of your choice, click the **New project** button.
3. Type a name for your project in the **Name** field, e.g. `psql-quickstart`.
4. In the **Prisma Postgres** section, click the **Get started** button.
5. In the **Region** dropdown, select the region that's closest to your current location, e.g. US East (N. Virginia).
6. Click the **Create project** button.

At this point, you'll be redirected to the Database page where you will need to wait for a few seconds while the status of your database changes from **PROVISIONING** to **CONNECTED**.

Once the green **CONNECTED** label appears, your database is ready to use!

## 3. Get your direct connection string

To connect to Prisma Postgres via direct TCP, you need to:

1. Open your project in the Prisma Console.
2. Navigate to your active Prisma Postgres instance.
3. Click the **API Keys** tab in the project's sidenav.
4. Click the **Create API key** button.
5. In the popup, provide a **Name** for the API key and click **Create**.
6. Copy the connection string starting with `postgres://`, this is your direct connection string.

## 4. Connect to your database

Use the connection string to connect to your Prisma Postgres database:

```terminal
psql "postgres://your-connection-string"
```

Alternatively, you can parse the connection string and use individual parameters:

```terminal
psql -h hostname -p port -U username -d database
```

Once connected, you'll see the psql prompt:

```
psql (16.0)
Type "help" for help.

database=>
```

## 5. Run your first SQL commands

Now you can start running SQL commands directly in the psql terminal.

### Create tables

```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  name TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE posts (
  id SERIAL PRIMARY KEY,
  title VARCHAR(255) NOT NULL,
  content TEXT,
  published BOOLEAN DEFAULT FALSE,
  author_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### Insert data

```sql
INSERT INTO users (email, name) 
VALUES ('alice@prisma.io', 'Alice');

INSERT INTO posts (title, content, published, author_id)
VALUES ('Hello World', 'This is my first post!', TRUE, 1);
```

### Query data

```sql
-- Select all users
SELECT * FROM users;

-- Select all posts with author information
SELECT 
  posts.id,
  posts.title,
  posts.published,
  users.name AS author_name,
  users.email AS author_email
FROM posts
JOIN users ON posts.author_id = users.id;

-- Find published posts
SELECT * FROM posts WHERE published = TRUE;
```

### Update data

```sql
UPDATE posts 
SET published = FALSE 
WHERE id = 1;
```

### Delete data

```sql
DELETE FROM posts WHERE id = 1;
```

## 6. Useful psql commands

psql provides many meta-commands (commands that start with `\`) to help you work with your database:

### List databases
```
\l
```

### Connect to a different database
```
\c database_name
```

### List tables
```
\dt
```

### Describe a table structure
```
\d table_name
```

### List all schemas
```
\dn
```

### Show current connection info
```
\conninfo
```

### Execute SQL from a file
```
\i /path/to/file.sql
```

### Toggle expanded display (useful for wide tables)
```
\x
```

### Get help
```
\?        -- List all psql commands
\h        -- List all SQL commands
\h SELECT -- Get help on a specific SQL command
```

### Quit psql
```
\q
```

## 7. Advanced features

### Execute a single command and exit

You can run a single SQL command without entering the interactive mode:

```terminal
psql "postgres://your-connection-string" -c "SELECT * FROM users;"
```

### Execute commands from a file

```terminal
psql "postgres://your-connection-string" -f script.sql
```

### Output results to a file

```terminal
psql "postgres://your-connection-string" -c "SELECT * FROM users;" -o output.txt
```

### Use CSV output format

```terminal
psql "postgres://your-connection-string" -c "SELECT * FROM users;" --csv
```

### Set environment variables

To avoid typing the connection string repeatedly, you can set environment variables:

```terminal
export PGHOST=hostname
export PGPORT=5432
export PGDATABASE=database
export PGUSER=username
export PGPASSWORD=password

# Now you can connect with just:
psql
```
