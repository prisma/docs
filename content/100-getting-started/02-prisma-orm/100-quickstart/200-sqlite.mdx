# Quickstart: Prisma ORM with SQLite

Get up and running with Prisma ORM and SQLite in under 15 minutes. SQLite is perfect for development, prototyping, and small applications as it requires no setup and stores data in a local file.

## Prerequisites

- [Node.js](https://nodejs.org/en/) installed on your machine (see [system requirements](/orm/reference/system-requirements) for officially supported versions)
- Basic knowledge of JavaScript or TypeScript

## 1. Create a new project

Create a new directory for your project and navigate into it:

```terminal
mkdir prisma-sqlite-quickstart
cd prisma-sqlite-quickstart
```

Initialize a new Node.js project:

```terminal
npm init -y
```

Install TypeScript and development dependencies:

```terminal
npm install typescript tsx @types/node --save-dev
```

Initialize TypeScript:

```terminal
npx tsc --init
```

## 2. Install and configure Prisma ORM

Install Prisma CLI and required packages:

```terminal
npm install prisma --save-dev
npm install @prisma/client @prisma/adapter-better-sqlite3
```

Initialize Prisma with SQLite:

```terminal
npx prisma init --output ../generated/prisma
```

This creates:
- A `prisma/` directory with a `schema.prisma` file
- A `.env` file for environment variables
- A `generated/prisma/` directory for the generated Prisma Client

## 3. Configure your schema for SQLite

Open `prisma/schema.prisma` and update it for SQLite with the modern generator:

```prisma file=prisma/schema.prisma
generator client {
  provider   = "prisma-client"
  output     = "../generated/prisma"
  engineType = "client"
}

datasource db {
  provider = "sqlite"
  url      = "file:./dev.db"
}

//add-start
model User {
  id    Int     @id @default(autoincrement())
  email String  @unique
  name  String?
  posts Post[]
}

model Post {
  id        Int     @id @default(autoincrement())
  title     String
  content   String?
  published Boolean @default(false)
  author    User    @relation(fields: [authorId], references: [id])
  authorId  Int
}
//add-end
```

## 4. Create your database

Create your first migration to set up the database:

```terminal
npx prisma migrate dev --name init
```

This command:
- Creates a `dev.db` SQLite file in your `prisma/` directory
- Creates the database tables based on your schema
- Generates the Prisma Client

## 5. Create a Prisma Client instance

Create a `lib/` directory and a `prisma.ts` file:

```typescript file=lib/prisma.ts
import { PrismaClient } from '../generated/prisma'
import { PrismaBetterSQLite3 } from '@prisma/adapter-better-sqlite3'

const adapter = new PrismaBetterSQLite3({ url: 'file:./prisma/dev.db' })

const globalForPrisma = globalThis as unknown as {
  prisma: PrismaClient | undefined
}

export const prisma = globalForPrisma.prisma ?? new PrismaClient({ adapter })

if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma
```

## 6. Write your first query

Create a `script.ts` file to test your setup:

```typescript file=script.ts
import { prisma } from './lib/prisma'

async function main() {
  // Create a new user with a post
  const user = await prisma.user.create({
    data: {
      name: 'Alice',
      email: 'alice@prisma.io',
      posts: {
        create: {
          title: 'Hello SQLite!',
          content: 'This is my first post with SQLite and Prisma ORM!',
          published: true,
        },
      },
    },
    include: {
      posts: true,
    },
  })
  console.log('Created user:', user)

  // Fetch all users with their posts
  const allUsers = await prisma.user.findMany({
    include: {
      posts: true,
    },
  })
  console.log('All users:', JSON.stringify(allUsers, null, 2))

  // Count total users
  const userCount = await prisma.user.count()
  console.log(`Total users: ${userCount}`)
}

main()
  .then(async () => {
    await prisma.$disconnect()
  })
  .catch(async (e) => {
    console.error(e)
    await prisma.$disconnect()
    process.exit(1)
  })
```

Run the script:

```terminal
npx tsx script.ts
```

You should see the created user and all users printed to the console!

## 7. Explore your data with Prisma Studio

Prisma Studio is a visual editor for your database. Launch it with:

```terminal
npx prisma studio
```

This opens a web interface where you can view and edit your SQLite data.
