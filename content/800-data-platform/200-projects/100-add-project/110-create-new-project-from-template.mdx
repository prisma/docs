<TopBlock>

If you want to explore how to create projects in the Prisma Data Platform, you can start with a template Prisma schema which is immediately migrated to an empty database.

</TopBlock>

## Prerequisites and considerations

* Log in to GitHub with your account ([or sign up for one](https://github.com/signup)).
* [Sign up for the Prisma Data Platform](/data-platform/account#sign-up-for-the-prisma-data-platform).
* Obtain a connection string to a [Prisma ORM-supported database](/reference/database-reference/supported-databases).
* [Grant the Prisma GitHub app read and write permissions to your GitHub repositories](/data-platform/projects/add-project#grant-read-and-write-permissions-to-the-prisma-github-app).

## Steps

1. Go to the [_Projects_](https://cloud.prisma.io/projects) page and click **New Project**.
2. On the _Configure project_ page, name your project and create a new GitHub repository.
    1. In **Display Name**, enter a name for your project.
        <Admonition>
        
        **Note**<br /><br />
        The display name of a project appears in the following places in the Prisma Data Platform:<br />
          &nbsp;• [_Projects_](https://cloud.prisma.io/projects) page<br />
          &nbsp;• [_Billing_](https://cloud.prisma.io/billing) page<br />
          &nbsp;• **Projects** drop-down in the upper left
        </Admonition>
    2. From **GitHub Accounts & Organizations**, select a GitHub account or an organization.
    3. Select **Create a repository**.
    4. (Optional) In **Repository Name**, enter a name for the repository.

        <Admonition type="info">
        
        **Note**<br /><br />

        • The GitHub repository name is pre-filled based on the display name, where the repository name is lowercase-only and spaces are replaced with hyphens.<br /><br />
        • You cannot create a project with a repository name duplicating that of an existing repository in the select account or organization.<br />
        </Admonition>

    5. Click **Next**.
        <img
        src="https://res.cloudinary.com/prismaio/image/upload/v1666700351/docs/pdp/create-manage-projects/create-project-01-configure-project_s4s19e.png"
        alt="Prisma Data Platform - Create new project - Name your project and create a new GitHub repository"
        width="750px"
        />
3. On the _Select template_ page, select an application template for your first project.
    1. Select one of the available application templates. When you select a template, you can see a preview of its Prisma schema file below.
        * **Blog**. Schema for a blog built with Next.js.
        * **Music Streaming Service**. Schema for a fictional music streaming service with a REST API.
        * **Rentals Platform**. Schema for a fictional hotel with a REST API.
        * **SaaS**. Schema for an identity service with a SaaS with a REST API.
        * **URL Shortener**. Schema for a URL shortener with a REST API.
        * **Empty**. An empty schema with no models, to build your own schema.

        <Admonition>
        
        **Note**<br /><br />

        If you plan to use PlanetScale or MongoDB, you can use only the **Empty** template to create a new project.
        </Admonition>
    2. (Optional) Clear **Include sample data** if you do not want to seed the database with sample data.

        <img
            src="https://res.cloudinary.com/prismaio/image/upload/v1666649988/docs/pdp/create-manage-projects/create-project-02-clear-include-sample-data_qvdps1.png"
            alt="Prisma Data Platform - Create a new project - Select a template project and seed your database"
            width="750px"
        />
    3. Click **Next**.
4. On the _Configure environment_ page, connect your database and configure the Data Proxy location.
    1. In **Connection string**, paste your database connection string.
        <Admonition type="alert">

        **Important**<br /><br />
        If you provide a connection string to a non-empty database (with existing tables and data), the project creation fails.<br /><br />

        To add a project with an existing database, see [Import an existing project with an existing database](/data-platform/projects/add-project/import-existing-project-existing-db).
        </Admonition>
    2. Under _Data Proxy_, from the **Location** drop-down menu, select the geographic location for the Data Proxy.
        <Admonition>
        
        **Note**<br /><br />
        Select a Data Proxy location that is close to the geographic location of your database.<br /><br />
        If you select a location that is remote from your database, then when your application sends requests to your database through the Data Proxy, you might notice some delay in the responses.
        </Admonition>
    3. From **Static IPs**, select **Enabled** if your database is behind a firewall and you can only configure external access from specific IP addresses. Copy the IP addresses and add them to the allowlist of your database.
        <Admonition>

        **Note**<br /><br />
        For more information, see [Static egress IP address](/data-platform/static-ips). 
        </Admonition>
    4. Click **Next**.
        <img
        src="https://res.cloudinary.com/prismaio/image/upload/v1666954777/docs/pdp/create-manage-projects/create-project-03-configure-database-connection-and-data-proxy_eeubnw.png"
        alt="Prisma Data Platform - Create a new project - Configure the database connection string and select a Data Proxy location"
        width="750px"
        />
5. On the _Deploy project_ page, you can deploy your project to Vercel, and save your connection strings and environment variables.
6. Click **Done**.
        
## Result

Based on the application template you selected, the **Data Browser** opens with a list of the models (tables or collections) in the database.
