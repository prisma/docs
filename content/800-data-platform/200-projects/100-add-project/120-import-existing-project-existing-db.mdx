<TopBlock>

Import your production application and configure the [Data Proxy](/data-platform/data-proxy) to enable connection pooling and avoid exhausting the database connections in serverless environments.

</TopBlock>

## Prerequisites and considerations

* Log in to GitHub with your account ([or sign up for one](https://github.com/signup)).
* Prepare a GitHub repository with your JavaScript/TypeScript application that uses Prisma ORM.
* Obtain a connection string to a [Prisma ORM-supported database](/reference/database-reference/supported-databases).
* [Sign up for the Prisma Data Platform](/data-platform/account#sign-up-for-the-prisma-data-platform).
* [Grant the Prisma GitHub app read and write permissions to your GitHub repositories](/data-platform/projects/add-project#grant-read-and-write-permissions-to-the-prisma-github-app).

## Steps

1. Go to the [_Projects_](https://cloud.prisma.io/projects) page and click **New Project**.
2. On the _Configure project_ page, name your project and select an existing GitHub repository.
    1. In **Display Name**, enter a name for your project.
        <Admonition>
        
        **Note**<br /><br />
        The display name of a project appears in the following places in the Prisma Data Platform:<br />
          &nbsp;• [_Projects_](https://cloud.prisma.io/projects) page<br />
          &nbsp;• [_Billing_](https://cloud.prisma.io/billing) page<br />
          &nbsp;• **Projects** drop-down in the upper left
        </Admonition>
    2. From **GitHub Accounts & Organizations**, select a GitHub account or an organization.
    3. Select **Import a Prisma repository**.
    4. From **Repository**, select the GitHub repository you want to import.
    5. From **Branch**, select the branch you want to import.
    6. In **Prisma Schema path**, enter the path to the `schema.prisma` file.
        <Admonition>
        
        **Note**<br /><br />

        The pre-filled path `prisma/schema.prisma` is the default path. Change it to the path you use for your project.
        </Admonition>
    7. Click **Next**.
        <img
        src="https://res.cloudinary.com/prismaio/image/upload/v1666699895/docs/pdp/create-manage-projects/import-project-01-configure-project_nt8fiz.png"
        alt="Prisma Data Platform - Import project with existing database - Name your project and select a GitHub repository and branch to import"
        width="750px"
        />
3. On the _Configure environment_ page, connect your database and configure the Data Proxy location.
    1. In the **Connection string** text box, paste your database connection string.
    2. Under _Data Proxy_, from the **Location** drop-down menu, select the geographic location for the Data Proxy.
        <Admonition>
        
        **Note**<br /><br />
        Select a Data Proxy location that is close to the geographic location of your database.<br /><br />
        If you select a location that is remote from your database, then when your application sends requests to your database through the Data Proxy, you might notice some delay in the responses.
        </Admonition>
    3. From **Static IPs**, select **Enabled** if your database is behind a firewall and you can only configure external access from specific IP addresses. Copy the IP addresses and add them to the allowlist of your database.
        <Admonition>

        **Note**<br /><br />
        For more information, see [Static egress IP address](/data-platform/static-ips). 
        </Admonition>
    4. Click **Create Project**.
        <img
        src="https://res.cloudinary.com/prismaio/image/upload/v1666954777/docs/pdp/create-manage-projects/create-project-03-configure-database-connection-and-data-proxy_eeubnw.png"
        alt="Prisma Data Platform - Import project with existing database - Configure the database connection string and select a Data Proxy location"
        width="750px"
        />

4. On the _Deploy project_ page, you can deploy your project to Vercel, and save your connection strings and environment variables.
5. Click **Done**.
        
## Result

The Data Browser opens with a list of the models (tables or collections) that exist in the database based on the application template you selected.
